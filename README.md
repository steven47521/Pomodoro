# Pomodoro 番茄钟（纯前端，可部署到 GitHub Pages）

一个**简洁、现代、纯前端**的番茄工作法（Pomodoro）计时器。  
无需后端、无需构建工具，直接将整个仓库推送到 GitHub 即可通过 GitHub Pages 在线使用。

---

## 功能概览

- **专注模式**：固定 25 分钟倒计时
- **休息模式**：固定 5 分钟倒计时
- **开始 / 暂停 / 重置** 控制
- **进度条展示**：当前轮进度一目了然
- **自动切换模式**：
  - 完成一轮专注后，自动切换到「休息」模式（不自动开始）
  - 完成一轮休息后，自动切换回「专注」模式（不自动开始）
- **轻量提示音**：每轮结束时播放短促提示音（如果浏览器支持 Web Audio）
- **自适应布局**：在电脑和手机上均有良好的展示效果

> 设计原则：界面尽量简洁、现代，不引入多余功能，方便初中生用户理解和修改。

---

## 目录结构

- `index.html`：页面入口文件，包含基础 HTML 结构和文案
- `style.css`：样式文件，负责整体视觉风格和响应式布局
- `main.js`：前端逻辑代码，实现计时器核心功能
- `LICENSE`：开源协议（如不需要，可按需修改或移除）

1. 将本仓库克隆或下载到本地：
   - 方法一：点击 GitHub 上的 **Code → Download ZIP**，解压到任意目录  
   - 方法二（熟悉 Git 时使用）：  
     ```bash
     git clone https://github.com/你的用户名/你的仓库名.git
     ```
2. 在资源管理器中双击打开 `index.html` 即可在浏览器中使用番茄钟。

> 提示：推荐使用最新版的 Chrome / Edge / Firefox 以获得更好的体验。

## 部署到 GitHub Pages

假设你已经在 GitHub 上创建了一个仓库，并把本项目代码放进去。

1. **推送代码到 GitHub**
   ```bash
   git add .
   git commit -m "Initial Pomodoro app"
   git push origin main
   ```
   如果你的默认分支叫 `master`，把上面命令中的 `main` 换成 `master`。

2. **开启 GitHub Pages**
   - 打开你的仓库页面，例如：`https://github.com/你的用户名/你的仓库名`
   - 点击顶部的 **Settings**
   - 左侧选择 **Pages**
   - 在 **Source** 中选择：
     - Branch：`main`（或 `master`）
     - Folder：`/ (root)`（即根目录）
   - 点击 **Save**

3. **访问你的在线番茄钟**
   - 几十秒到几分钟后，你会在 Pages 页面看到一个访问链接，大致形如：  
     `https://你的用户名.github.io/你的仓库名/`
   - 打开这个链接，就可以在任何地方使用你的番茄钟应用。

## 代码说明

### `index.html`

- 定义了整个应用的结构：
  - 标题和说明文案
  - 模式切换按钮（专注 / 休息）
  - 时间显示区域
  - 进度条
  - 控制按钮（开始 / 暂停 / 重置）
- 通过 `<link rel="stylesheet" href="./style.css" />` 引入样式  
  以及 `<script src="./main.js"></script>` 引入计时逻辑。

### `style.css`

- 以深色渐变背景 + 高光色为主，营造专注氛围
- 使用 CSS 变量管理颜色、圆角、阴影等，便于后续整体调整风格
- 对按钮、进度条、卡片布局进行了细致设计，在桌面端和移动端都有良好观感

### `main.js`

- 使用一个自执行函数 `(function () { ... })()` 包裹所有代码，避免变量泄露到全局
- 主要逻辑：
  - 使用 `setInterval` 每秒更新倒计时
  - 剩余时间为 0 时停止计时、播放提示音、更新状态文案，并**自动切换**模式
  - 支持开始、暂停、重置操作，并根据状态控制按钮是否可点击
  - 通过简单的数学计算更新进度条宽度
- 声音：
  - 使用 Web Audio API 生成一个约 0.4 秒的短促“滴”声
  - 如果浏览器不支持相关 API，代码会静默失败，不影响计时功能

## 常见修改点（给不会写代码的你）

你可以只修改最简单的部分，来满足自己的习惯。

- **修改专注 / 休息时间**
  - 打开 `main.js`
  - 找到：
    ```js
    const MODES = {
      focus: {
        label: "专注",
        duration: 25 * 60, // 25 分钟
      },
      break: {
        label: "休息",
        duration: 5 * 60, // 5 分钟
      },
    };
    ```
  - 把 `25` 或 `5` 改成你想要的分钟数即可，例如 `50 * 60` 表示 50 分钟。

- **关闭提示音**
  - 在 `main.js` 中找到 `playNotificationSound();`
  - 在这行前面加上 `//`，变成注释即可：
    ```js
    // playNotificationSound();
    ```

- **修改界面文案**
  - 大部分中文文案都直接写在 `index.html` 或 `main.js` 里，你可以用编辑器搜索中文内容然后修改。

如需增加进阶功能（如：统计完成轮数、长休息模式、自定义主题等），可以在当前结构基础上继续扩展。

## 维护与后续改进建议

本项目刻意保持了**前端单页面 + 纯静态文件**的极简结构，适合在学习阶段理解和改造。  
如果未来你的需求变复杂（例如需要账号、云同步统计数据等），可以考虑：

- 使用框架（如 React / Vue）重构为组件化应用
- 引入状态管理（如 Redux、Pinia）统计番茄钟历史数据
- 接入简单的后端服务或 Serverless 存储完成记录

在那之前，这个版本已经足够作为一个轻量、可靠的个人番茄钟工具。
